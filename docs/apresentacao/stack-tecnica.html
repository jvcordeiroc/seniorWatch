O server é a fonte da verdade dos contratos; o tutor mantém um espelho na fronteira de rede — uma duplicação consciente que desacopla o build do frontend do backend.

A interface

Os contratos congelam tudo

contracts.ts
interface Telemetry {
  deviceId: string;
  timestamp: string;        // ISO-8601
  heartRate: number;        // BPM
  spo2: number;             // saturação de O₂
  battery: number;
  location: { lat: number; lng: number; accuracy: number };
  fallDetected: boolean;
}

type AlertType = 'fall' | 'high_hr' | 'low_hr' | 'low_battery' | 'offline';

// é exatamente isto que o tutor recebe pelo WebSocket:
type StreamEvent = TelemetryEvent | AlertEvent;

É este contrato que o relógio nativo vai implementar na Fase 2 — sem tocar no backend nem no app.

Backend

Unidades pequenas e testáveis

alert-engine
Função pura: recebe uma leitura, devolve alertas. Cooldown anti-spam, sem I/O.
telemetry-store
Persistência e consulta em node:sqlite (histórico, último, lastSeen).
realtime-hub
Gerencia conexões WS e faz broadcast de telemetria e alertas.
validation
Schema Zod; rejeita payload inválido com 400 antes de persistir.
server
Fábrica do Fastify: rotas REST + endpoint WebSocket, injetável em teste.
index
Bootstrap + verificador periódico de offline.

server.ts
app.post('/v1/telemetry', async (req, reply) => {
  const t = parseTelemetry(req.body);          // Zod
  store.insert(t);
  hub.broadcast({ type: 'telemetry', data: t });
  for (const a of engine.evaluate(t, Date.now())) hub.broadcast(a);
});

Em dev, o Vite faz proxy de /v1 (incluindo o upgrade do WebSocket) para o backend — o navegador fala numa porta só.

Decisões técnicas

Escolhas que valem explicar

1

node:sqlite via process.getBuiltinModule

O módulo é experimental e não consta em builtinModules sem o prefixo, o que quebra o bundler (Vitest/Vite). Carregá-lo em runtime o torna invisível ao empacotador; os tipos vêm de um import type.

2

Zero dependências nativas

SQLite, fetch e WebSocket são todos nativos do Node 25 — instalação simples, sem compilação de binários.

3

Contratos espelhados na fronteira

Duplicar os tipos type-only entre server e tutor desacopla o build do frontend do backend — o trade-off é manter os dois em sincronia.

4

Motor de alertas como função pura

Sem I/O: a lógica de regras e cooldown é testável exaustivamente sem subir servidor nem banco.
